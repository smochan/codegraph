## 🔴 Things only you can do

### 1. Push the `v0.1.0` tag → triggers the release

```bash
cd <wherever you cloned codegraph>
git push origin v0.1.0
```

What happens:
- The `release.yml` workflow runs on GitHub.
- It builds wheel + sdist, validates with `twine check`.
- It creates a **GitHub Release** at `https://github.com/smochan/codegraph/releases/tag/v0.1.0`
  using `CHANGELOG.md` as the body and attaching `dist/*` for download.
- If `PYPI_API_TOKEN` is set (see step 2), it also publishes to PyPI.

Verify:
```bash
gh run list --workflow=release.yml --limit 1
gh release view v0.1.0
```

### 2. (Optional) Publish to PyPI

You need a PyPI account and a project-scoped API token.

1. Create the token: https://pypi.org/manage/account/token/
   - First time: scope `Entire account`. After the first upload you can
     scope subsequent tokens to the `codegraph-py` project.
2. Add it to the repo:
   ```bash
   gh secret set PYPI_API_TOKEN --repo smochan/codegraph
   ```
   Paste the token when prompted (it starts with `pypi-`).
3. If you've already pushed the tag without the secret, re-run the
   workflow:
   ```bash
   gh workflow run release.yml --ref v0.1.0
   ```

Verify:
- The package shows up at https://pypi.org/project/codegraph-py/
- `pip install codegraph-py` works from any machine.

### 3. Move the project to your personal computer

Everything that matters is on GitHub `main`. On the new machine:

```bash
git clone https://github.com/smochan/codegraph.git
cd codegraph
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -e ".[dev]"
pytest -q                  # expect: 130 passed
codegraph --help           # expect: subcommand list
```

Things that DO NOT come over (regenerable):
- `.codegraph/graph.db` — generated by `codegraph build`
- `.codegraph/explore/` — generated by `codegraph serve`
- `.venv/` — recreate on the new machine
- the local `v0.1.0` tag — only present if you've already pushed it (step 1)

### 4. Test the project end-to-end on your machine

```bash
# Build the graph for codegraph itself
codegraph build

# Run analysis
codegraph analyze

# Open the web dashboard (http://127.0.0.1:8765)
codegraph serve

# Try the MCP server
codegraph mcp --help
codegraph mcp serve --db .codegraph/graph.db
# (Ctrl-C to stop)

# Try the PR review pipeline
codegraph baseline save
# ...edit some files...
codegraph build
codegraph review --format md
```

Want to point codegraph at a *different* repo? Just `cd` into that repo
and run the same commands — `codegraph build` infers the root from
`cwd`.

### 5. Wire codegraph into Claude Code

Add to `~/.claude.json` (or the equivalent for your client):

```json
{
  "mcpServers": {
    "codegraph": {
      "command": "codegraph",
      "args": ["mcp", "serve", "--db", ".codegraph/graph.db"]
    }
  }
}
```

Restart Claude Code, then in any conversation try:

> "What functions have the highest blast radius in this repo?"
> "Show me callers of `build_dashboard_payload`."
> "Are there any cycles between `codegraph.analysis` and `codegraph.viz`?"

- Go, Java, Rust, C#, Ruby, PHP.
- Each is mostly a copy of `codegraph/parsers/python.py` with the
